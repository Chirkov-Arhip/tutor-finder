# tutor-finder

Web-приложение для онлайн-подбора репетиторов

TutorFinder — web-приложение для онлайн-подбора репетиторов, ориентированное на школьников и студентов. Ученики могут находить репетиторов по предметам и отправлять им заявки. Репетиторы управляют входящими заявками и принимают или отклоняют их.

## Функции приложения

- Регистрация и вход в аккаунт
- Два типа пользователей - ученик и репетитор
- Заполнение профиля с личными данными
- Поиск репетиторов с фильтрацией по предметам
- Отправка заявки репетитору с указанием предмета
- Принятие и отклонение заявок репетитором
- Обмен контактами после принятия заявки
- Редактирование профиля с подтверждением паролем
- Смена пароля
- Обращение в техподдержку с отправкой письма на почту

## Установка и запуск приложения локально

### Предварительные требования

- Python 3.11+
- PostgreSQL 14+
- Git

### Шаги установки

1. Клонируйте репозиторий:

```bash
git clone https://github.com/Chirkov-Arhip/tutor-finder.git
cd tutor-finder
```

2. Убедитесь, что у вас установлен Python 3.11 или новее, а также PostgreSQL (сервер должен быть запущен).

3. Установите Poetry (если ещё не установлен):

```bash
pip install poetry
```

4. Установите зависимости проекта:

```bash
poetry install
```

5. Создайте файл `.env` в корне проекта на основе `.env.example` и заполните его:

```text
DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost/tutor_finder
SECRET_KEY=ваш_секретный_ключ
MAIL_USERNAME=ваша_почта@yandex.ru
MAIL_PASSWORD=пароль_приложения
MAIL_TO=ваша_почта@yandex.ru
```

6. Запустите приложение:

```bash
poetry run start
```

База данных и все таблицы создадутся автоматически при первом запуске.
Никаких дополнительных действий (создание БД, ручной запуск миграций) не требуется.

Приложение будет доступно по адресу http://127.0.0.1:8080

Документация API в формате Swagger доступна по адресу http://127.0.0.1:8080/docs

## Организация работы с зависимостями проекта

В процессе разработки приложения часто требуется использовать сторонние библиотеки. Управление зависимостями позволяет:

- Гарантировать одинаковые версии библиотек у всех участников команды.
- Автоматизировать установку — не нужно вручную скачивать каждую зависимость.
- Обеспечить воспроизводимость окружения на разных компьютерах.

### Что такое Poetry?

Poetry — современный менеджер зависимостей для Python. Он позволяет:

- Описывать зависимости в одном файле (`pyproject.toml`)
- Устанавливать, обновлять и удалять библиотеки одной командой
- Изолировать зависимости в виртуальном окружении

### Основные файлы Poetry

- `pyproject.toml` — основной файл конфигурации. В нём указываются все зависимости, настройки и скрипты запуска.
- `poetry.lock` — содержит точные версии всех установленных библиотек.

### Основные команды Poetry

Установка всех зависимостей:

```bash
poetry install
```

Добавление новой библиотеки:

```bash
poetry add requests
```

Удаление библиотеки:

```bash
poetry remove requests
```

Обновление зависимостей:

```bash
poetry update
```

### Рекомендации

- Не редактируйте вручную `pyproject.toml` и `poetry.lock` — используйте только команды Poetry.
- Всегда выполняйте `poetry install` после получения новых изменений из репозитория.
- Если добавили или удалили библиотеку — закоммитьте изменения в `pyproject.toml` и `poetry.lock`.

## Конфигурирование приложения и работа с переменными окружения

### Файл .env

Файл `.env` — текстовый файл в корне проекта с секретными настройками. Создайте его на основе `.env.example`:

```text
DATABASE_URL=postgresql://postgres:пароль@localhost/tutor_finder
SECRET_KEY=секретный_ключ
MAIL_USERNAME=почта@yandex.ru
MAIL_PASSWORD=пароль_приложения
MAIL_TO=почта@yandex.ru
```

> **Важно:** Файл `.env` добавлен в `.gitignore` и не попадает в репозиторий.

В проекте используется библиотека `pydantic-settings`, которая автоматически подгружает переменные из `.env`. Все параметры описаны в `src/config.py`.

### Почему нельзя хранить секретные данные в исходном коде?

- **Безопасность:** Пароли в публичном репозитории доступны всем.
- **Гибкость:** Разные окружения (разработка, продакшн) требуют разных настроек.
- **Командная работа:** Каждый разработчик использует свои параметры.

### Рекомендации по работе с переменными

- Все чувствительные параметры выносите в `.env`.
- Не добавляйте `.env` в репозиторий.
- Для новых параметров добавьте их в `.env` и опишите в `src/config.py`.

## Миграции базы данных

### Зачем нужны миграции?

Миграции позволяют безопасно изменять структуру базы данных. Благодаря им можно:

- Изменять структуру БД без потери данных
- Откатывать изменения при ошибках
- Синхронизировать структуру БД между всеми разработчиками

### Alembic в этом проекте

В проекте используется Alembic, интегрированный с SQLAlchemy. Все миграции хранятся в папке `migrations/versions`.

### Автоматическое применение миграций

При запуске приложения (через `poetry run start` или `poetry run dev`) Alembic автоматически применяет все новые миграции. Вам не нужно вручную выполнять `alembic upgrade head`.
База данных также создаётся автоматически при её отсутствии.

### Как изменить структуру данных и создать миграцию?

1. Измените модели в папке `src/db/models`.

2. Сгенерируйте миграцию:

```bash
poetry run alembic revision --autogenerate -m "описание изменения"
```

3. Примените миграцию (просто перезапустите приложение, либо выполните вручную):

```bash
poetry run alembic upgrade head
```

### Полезные команды Alembic

```bash
# Создать миграцию
poetry run alembic revision --autogenerate -m "описание"

# Применить миграции (если не хотите перезапускать сервер)
poetry run alembic upgrade head

# Откатить последнюю миграцию
poetry run alembic downgrade -1

# Текущий статус
poetry run alembic current

# История миграций
poetry run alembic history
```

### Рекомендации по работе с миграциями

- После изменения моделей всегда создавайте и применяйте миграцию.
- Не редактируйте файлы миграций вручную.
- Все миграции должны быть в репозитории.
