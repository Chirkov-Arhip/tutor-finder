# tutor-finder-api

Серверная часть web-приложения для онлайн-подбора репетиторов

TutorFinder — web-приложение для онлайн-подбора репетиторов. Ученики могут находить репетиторов по предметам и отправлять им заявки. Репетиторы управляют входящими заявками и принимают или отклоняют их.

## Функции приложения

- Регистрация и вход в аккаунт
- Два типа пользователей — ученик и репетитор
- Заполнение профиля с личными данными
- Поиск репетиторов с фильтрацией по предметам
- Отправка заявки репетитору с указанием предмета
- Принятие и отклонение заявок репетитором
- Обмен контактами после принятия заявки
- Редактирование профиля с подтверждением паролем
- Смена пароля
- Обращение в техподдержку с отправкой письма на почту

## Установка и запуск приложения локально

### Предварительные требования

- Python 3.11+
- PostgreSQL 14+
- Git

### Шаги установки

1. Клонируйте репозиторий:

```bash
git clone https://github.com/Chirkov-Arhip/tutor-finder-api.git
cd tutor-finder-api
```

2. Создайте виртуальное окружение:

```bash
python -m venv venv
```

3. Активируйте виртуальное окружение:

```bash
# Windows
venv\Scripts\activate

# Linux / macOS
source venv/bin/activate
```

4. Установите Poetry:

```bash
pip install poetry
```

5. Установите зависимости проекта:

```bash
poetry install
```

6. Создайте файл `.env` в корне проекта и заполните его:

```
DATABASE_URL=postgresql://postgres:ВАШ_ПАРОЛЬ@localhost/tutor_finder
SECRET_KEY=ваш_секретный_ключ
MAIL_USERNAME=ваша_почта@yandex.ru
MAIL_PASSWORD=пароль_приложения
MAIL_TO=ваша_почта@yandex.ru
```

7. Создайте базу данных `tutor_finder` в PostgreSQL.

8. Примените миграции:

```bash
poetry run alembic upgrade head
```

9. Запустите приложение:

```bash
poetry run start
```

Документация API в формате Swagger доступна по адресу [http://127.0.0.1:8080/docs](http://127.0.0.1:8080/docs)

## Организация работы с зависимостями проекта

### Зачем нужно управление зависимостями?

В процессе разработки приложения часто требуется использовать сторонние библиотеки (например, FastAPI, Uvicorn и др.). Управление зависимостями позволяет:

- **Гарантировать, что у всех участников команды одинаковые версии библиотек** — это снижает вероятность ошибок, связанных с несовместимостью версий.
- **Автоматизировать установку и обновление библиотек** — не нужно вручную скачивать и устанавливать каждую зависимость.
- **Обеспечить воспроизводимость окружения** — приложение будет одинаково работать на разных компьютерах и серверах.

### Что такое Poetry?

[Poetry](https://python-poetry.org/) — это современный менеджер зависимостей и инструмент для управления проектами на Python. Он позволяет:

- Описывать зависимости проекта в одном файле (`pyproject.toml`)
- Устанавливать, обновлять и удалять библиотеки одной командой
- Изолировать зависимости проекта в виртуальном окружении

### Основные файлы Poetry

- **`pyproject.toml`** — основной файл конфигурации проекта. В нём указываются все зависимости, настройки проекта и скрипты запуска.
- **`poetry.lock`** — автоматически создаётся Poetry и содержит точные версии всех установленных библиотек. Благодаря этому у всех разработчиков будут одинаковые версии библиотек.

### Основные команды Poetry

Установка всех зависимостей проекта (после клонирования репозитория):

```bash
poetry install
```

Добавление новой библиотеки (например, requests):

```bash
poetry add requests
```

Удаление библиотеки (например, requests):

```bash
poetry remove requests
```

Обновление всех зависимостей до последних разрешённых версий:

```bash
poetry update
```

Запуск команд внутри виртуального окружения Poetry:

```bash
poetry run <команда>
# Например:
poetry run python main.py
```

### Рекомендации

- Не редактируйте вручную файлы `pyproject.toml` и `poetry.lock` — используйте только команды Poetry.
- Всегда выполняйте `poetry install` после получения новых изменений из репозитория, чтобы установить новые зависимости.
- Если добавили или удалили библиотеку — обязательно закоммитьте изменения в `pyproject.toml` и `poetry.lock`.

Подробнее о Poetry можно узнать в [официальной документации](https://python-poetry.org/docs/).

## Конфигурирование приложения и работа с переменными окружения

Для гибкой настройки приложения используются переменные окружения и файл `.env`. Это позволяет менять параметры работы без изменения исходного кода.

### Что такое переменные окружения?

Переменные окружения — это специальные значения, которые задаются в операционной системе и могут быть использованы приложением во время выполнения. Например, можно задать имя пользователя и пароль для подключения к базе данных, не прописывая их в коде.

### Файл .env

Файл `.env` — текстовый файл в корне проекта, в котором хранятся пары ключ=значение:

```
DATABASE_URL=postgresql://postgres:пароль@localhost/tutor_finder
SECRET_KEY=секретный_ключ
MAIL_USERNAME=почта@yandex.ru
MAIL_PASSWORD=пароль_приложения
MAIL_TO=почта@yandex.ru
```

**Важно:** Файл `.env` добавлен в `.gitignore` — он не попадает в репозиторий, так как содержит секретные данные.

В проекте используется библиотека `pydantic-settings`, которая автоматически подгружает переменные из `.env`. Все параметры описаны в файле `src/config.py`.

### Почему нельзя хранить секретные данные в исходном коде?

- **Безопасность:** Если пароли и ключи попадут в публичный репозиторий, злоумышленники смогут получить доступ к базе данных.
- **Гибкость:** Для разных окружений (разработка, продакшн) удобно использовать разные настройки, не меняя код.
- **Командная работа:** Каждый разработчик может использовать свои параметры, не мешая другим.

### Как задать переменные окружения вручную

Можно задать переменные окружения прямо в терминале перед запуском приложения:

```bash
export DATABASE_URL=postgresql://postgres:пароль@localhost/tutor_finder
poetry run start
```

### Рекомендации по работе с переменными

- Все чувствительные и изменяемые параметры выносите в `.env` или переменные окружения.
- Не добавляйте `.env` в репозиторий.
- Для новых параметров добавьте их в `.env` и опишите в классе `Settings` в `src/config.py`.

## Миграции базы данных

### Зачем нужны миграции?

В процессе разработки структура базы данных часто меняется: добавляются новые сущности, изменяются существующие, удаляются устаревшие поля. Миграции позволяют автоматически и безопасно применять такие изменения. Благодаря миграциям можно:

- Изменять структуру БД без потери данных
- Легко откатывать изменения при ошибках
- Синхронизировать структуру БД между всеми разработчиками

### Как это работает?

Миграции — это специальные Python-скрипты, которые создаются автоматически с помощью инструмента Alembic. Каждый такой скрипт содержит инструкции как изменить структуру базы данных. Alembic отслеживает какие миграции уже применены и позволяет применять новые или откатывать старые.

### Alembic в этом проекте

В проекте используется библиотека [Alembic](https://alembic.sqlalchemy.org/), которая интегрируется с SQLAlchemy. Все миграции хранятся в папке `migrations/versions`.

### Как изменить структуру данных и создать миграцию?

1. **Измените модели данных** в папке `src/db/models`.

2. **Сгенерируйте новую миграцию:**

```bash
poetry run alembic revision --autogenerate -m "описание изменения"
```

Alembic сравнит текущие модели и структуру БД, и создаст скрипт миграции.

3. **Примените миграцию к базе данных:**

```bash
poetry run alembic upgrade head
```

### Полезные команды Alembic

Создать новую миграцию (автоматически):

```bash
poetry run alembic revision --autogenerate -m "описание"
```

Применить все новые миграции:

```bash
poetry run alembic upgrade head
```

Откатить последнюю миграцию:

```bash
poetry run alembic downgrade -1
```

Посмотреть текущий статус миграций:

```bash
poetry run alembic current
```

Показать историю миграций:

```bash
poetry run alembic history
```

### Рекомендации по работе с миграциями

- После изменения моделей всегда создавайте и применяйте миграцию.
- Не редактируйте файлы миграций вручную, если не уверены в своих действиях.
- Все миграции должны быть в репозитории, чтобы все разработчики работали с одной структурой данных.
